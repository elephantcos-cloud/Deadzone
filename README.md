# Dead Zone Mirror

স্ক্রিনের নষ্ট/কালো অংশ এড়িয়ে পুরো স্ক্রিন একটা ছোট, নিজের ইচ্ছেমতো বসানো ও আকার দেওয়া overlay-তে দেখা এবং ব্যবহার করার জন্য একটা ইউটিলিটি অ্যাপ।

## কীভাবে কাজ করে
1. **MediaProjection** দিয়ে পুরো স্ক্রিনের রিয়েল-টাইম ছবি ক্যাপচার হয়।
2. সেই ছবি একটা **floating overlay window**-এ (ভালো/কাজ-করা অংশে) ছোট করে দেখানো হয়।
3. Overlay-তে করা টাচ, **AccessibilityService** দিয়ে আসল স্ক্রিনের ঠিক সেই জায়গায় (নষ্ট অংশ হলেও) সিন্থেটিক gesture হিসেবে পাঠানো হয়।

## ইনস্টলের পর করণীয় (একবারই)
এই তিনটা ধাপ ছাড়া অ্যাপ কাজ করবে না — কোড বাগ না, এগুলো Android-এর নিরাপত্তা-সংক্রান্ত ম্যানুয়াল অনুমতি, অ্যাপ নিজে থেকে চালু করতে পারে না:

1. **Overlay Permission** — অ্যাপ খুলে "Permission দাও" চাপো, "Display over other apps" অন করো।
2. **Accessibility Service** — "চালু করো" চাপলে Settings খুলবে। Termux দিয়ে সরাসরি ইনস্টল করা অ্যাপ হওয়ায় (Android 13+) toggle প্রথমে ধূসর থাকতে পারে — সেক্ষেত্রে App Info-তে গিয়ে উপরে ৩-ডট মেনু থেকে **"Allow restricted settings"** অন করে আবার Accessibility থেকে চালু করো।
3. **Battery optimization / Autostart** — Settings > Battery > এই অ্যাপকে "Don't optimize" / autostart-এ allow করে রাখো, নাহলে Realme UI মাঝেমধ্যে ব্যাকগ্রাউন্ড overlay বন্ধ করে দিতে পারে।

## ব্যবহার
- "Mirror শুরু করো" চাপলে স্ক্রিন-রেকর্ডিং পারমিশনের সিস্টেম ডায়ালগ আসবে — Start চাপো। (নতুন Android ভার্সনে এটা প্রতিবার চালু করলেই আবার চাইতে পারে, এটা Android-এর নিয়ম, অ্যাপের সমস্যা না।)
- Overlay-র উপরে-ডানে ছোট **lock বাটন** চেপে Edit mode / Lock mode বদলানো যায়:
  - **Edit mode:** overlay যেকোনো জায়গায় ধরে টানলে সরে, নিচে-ডানে কোণা ধরে টানলে আকার বদলায়।
  - **Lock mode:** overlay-তে করা টাচ সরাসরি আসল স্ক্রিনে রিডাইরেক্ট হয় — এটাই সাধারণ ব্যবহারের মোড।
- যেখানে/যে সাইজে শেষবার রাখা হয়েছিল, পরের বার চালু করলে সেখানেই খুলবে (আলাদা করে সেট করতে হবে না)।

## জানা সীমাবদ্ধতা
- Overlay যেখানে বসানো, ঠিক সেই ছোট অংশটুকু মিরর-এ একটা সলিড ধূসর প্যাচ হিসেবে দেখাবে (ইচ্ছাকৃত — নিজেই নিজেকে বারবার দেখানো/infinite mirror ঠেকাতে)। বাকি পুরো স্ক্রিন স্বাভাবিকভাবে মিরর হবে।
- স্ক্রিন ঘোরালে (rotate) ক্যাপচার নিজে থেকেই নতুন মাপে রিফ্রেশ হয়, তবে সংক্ষিপ্ত একটা মুহূর্তের জন্য ছবি ফ্রিজ থাকতে পারে।
- খুব দ্রুত swipe/scroll করলে, আসল স্ক্রিনে সেই gesture আঙুল ওঠানোর পরই (পুরো path হিসাব করে) পাঠানো হয় — তাই সামান্য দেরি (gesture-এর দৈর্ঘ্য অনুযায়ী) স্বাভাবিক।
- Build হয় GitHub Actions দিয়ে (`.github/workflows/build.yml`) — Gradle wrapper বান্ডেল করা হয়নি, workflow নিজেই Gradle 8.4 সেটআপ করে নেয়, তাই কোনো PC/local Gradle লাগে না।
